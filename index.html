国立国会図書館サーチ API

各機関のアプリケーション等から「国立国会図書館サーチ」を検索し、メタデータを取得するための 外部提供インタフェース仕様を記述した公式ドキュメントです。
すべての API のベース URL は https://ndlsearch.ndl.go.jp です。

バージョン: 第 1.3 版
最終更新: 2025.3.26
文字コード: UTF-8
プロトコル: HTTPS（HTTP は自動リダイレクト）

データプロバイダ・データグループ

検索対象データベース（データプロバイダ）はデータプロバイダ ID で指定します。データグループを指定することで、複数のデータプロバイダを横断して検索することも可能です。最新の一覧は附録１・附録２を参照してください。

📌
書影 API はデータプロバイダ・データグループの指定はありません。

mediaType

SRU・OpenSearch・OpenURL では mediatype パラメータによって資料の種別・形態を絞り込めます。最新の mediaType 一覧は附録２（「データグループ ID・mediaType 一覧」）を参照してください。OAI-PMH・書影 API では指定できません。

文字コード

すべてのインタフェースで文字コードは UTF-8 です。

返戻データ形式（DC-NDL / dcndl_v3）

返戻データに含まれる dcndl スキーマは「国立国会図書館ダブリンコアメタデータ記述（DC-NDL）」に基づき本サービスで定義した形式です。詳細は DC-NDL スキーマ を参照してください。

⚠️
dcndl_v3 は「DC-NDL（RDF）フォーマット仕様 ver.3.0」（2025年4月ドラフト版公開予定）に基づいた形式ですが、正式版ではないことに留意してください。

内部 JSON の参照

本サービスの内部項目である JSON ファイルは以下のリクエスト形式で確認できます。

GET https://ndlsearch.ndl.go.jp/api/bib/external/search?cs=bib&f-token=[リポジトリ番号-アイテム番号]
リクエスト例
https://ndlsearch.ndl.go.jp/api/bib/external/search?cs=bib&f-token=R100000039-I1000732
API 1

SRU（Search/Retrieve Via URL）

Z39.50 を発展させた、REST を利用して情報内容を検索するプロトコルです。本サービスでは SRU バージョン 1.1 および 1.2 をサポートします（ZEEREX はサポートしません）。

基本仕様: http://www.loc.gov/standards/sru/

リクエスト形式

GET https://ndlsearch.ndl.go.jp/api/sru?operation=searchRetrieve&recordSchema=[schema]&query=[CQL]
ℹ️
サポートするオペレーションは searchRetrieve のみです（explain・scan はサポートしません）。

searchRetrieve の引数

No. 引数名 必須/任意 説明
1 operation 必須 "searchRetrieve" を指定
2 version 任意 1.1 または 1.2。省略時は 1.2 とみなす
3 query 必須 検索条件（CQL）。詳細は CQL 仕様 を参照
4 startRecord 任意 取得開始位置。省略時は 1
5 maximumRecords 任意 最大取得件数（最大 500 件）。省略時は 200
6 recordPacking 任意 "xml" または "string"。省略時は "string"
7 recordSchema 任意 dc または dcndl（省略時は dc）。dcndl_v3（ドラフト版）も指定可能
10 sortKeys 任意 ソートキー情報。version=1.1 のときのみ有効
12 path 任意 title / creator / created_date / modified_date。version=1.1 かつ sortKeys 指定時のみ有効
13 ascending 任意 "0"（降順）/ "1"（昇順・デフォルト）。version=1.1 かつ sortKeys 指定時のみ有効
16 inprocess 任意 "true" を指定すると国立国会図書館新着書誌情報のみを取得
17 onlyBib 任意 "true" を指定すると書誌情報のみを取得。recordSchema=dcndl 指定時のみ有効
📌
返戻件数の制約： startRecord と maximumRecords を指定しても、501 件目以降は取得できません（検索負荷回避のための制約）。

CQL 仕様

CQL 1.2 を基本とします。クエリは AND・OR で連結できます。

⚠️
検索キーワードに AND・OR（andy・organic なども含む）が含まれる場合は検索エラーになります。この場合は %3d の前後に %20 を付与してください。

例: title%20%3d%20%22andy%22

検索項目一覧

No. 参照名 内容 一致条件 論理条件 複数値
1 dpid データプロバイダ ID、データグループ ID、コレクションコード、AccessRights 完全一致 = / any ○（OR）
3 title タイトル 部分一致 ○ ○（AND）
4 creator 作成者 部分一致 ○ ○（AND）
5 publisher 出版者 部分一致 ○ ○（AND）
6 digitized_publisher デジタル化した製作者 部分一致 ○ ○（AND）
7 ndc 分類（NDC、NDLC、LCC、DDC、UDC） 前方一致 = のみ ×
8 ndlc 分類（NDLC） 前方一致 = のみ ×
9 description 内容記述 部分一致 ○ ○（AND）
10 subject 主題 部分一致 ○ ○（AND）
11 isbn ISBN（10桁・13桁どちらでも検索可能） 完全一致 = のみ ×
12 issn ISSN 完全一致 = のみ ×
13 jpno 全国書誌番号 完全一致 = のみ ×

19 sortBy ソート基準（version=1.2 指定時のみ利用可）。title / creator / issued_date / created_date / modified_date に /sort.ascending か /sort.descending を続けて指定 — — —
💡
from と until を両方指定する場合、指定フォーマットを揃える必要があります（例: from=YYYY-MM なら until=YYYY-MM）。
同一項目に複数値を指定する際は all（AND）・any（OR）を使って簡潔に記述できます。
例: creator all 夏目 芥川（= creator=夏目 AND creator=芥川）

API 2

OpenSearch

A9.com が提唱した横断検索プロトコル。URL パラメータで検索リクエストを発行し、RSS 2.0 形式で結果を取得します。本サービスの対応バージョンは 1.0 です。

基本仕様: https://github.com/dewitt/opensearch

リクエスト形式

GET https://ndlsearch.ndl.go.jp/api/opensearch?dpid=[dpid]&title=[title]&…

引数一覧

No.パラメータ名内容一致条件複数指定
1dpidデータプロバイダ ID、データグループ ID、コレクションコード、AccessRights完全一致○（OR）
2dpgroupidデータプロバイダグループ ID完全一致×
3anyすべての項目を対象に検索部分一致○
4titleタイトル部分一致○
5creator作成者部分一致○
6publisher出版者部分一致○
7digitized_publisherデジタル化した製作者部分一致○
8ndc分類（NDC、NDLC）前方一致×
9from開始出版年月日（YYYY/YYYY-MM/YYYY-MM-DD）—×
10until終了出版年月日（YYYY/YYYY-MM/YYYY-MM-DD）—×
11cnt出力レコード上限値（最大 500 件、省略時 200）—×
12idxレコード取得開始位置（省略時は 1）—×
13isbnISBN（10 桁・13 桁入力時は両方に変換して完全一致。それ以外は前方一致）完全一致 or 前方一致×
14mediatype資料種別（附録２参照）完全一致○（OR）
ℹ️
項目間の論理条件はすべて AND です。
一つの検索項目内に半角スペースで複数キーワードを指定できます（dpid・mediatype は OR 検索、それ以外は AND 検索）。
dpid のみを指定した検索はできません。
引数の誤りがあった場合は検索結果 0 件になります（エラーは返戻されません）。

リクエスト例

例1: タイトルに「マリーアントワネット」を含むもの

リクエスト URL
https://ndlsearch.ndl.go.jp/api/opensearch?title=%e3%83%9e%e3%83%aa%e3%83%bc%e3%82%a2%e3%83%b3%e3%83%88%e3%83%af%e3%83%8d%e3%83%83%e3%83%88

例2: 青空文庫の中で著者に「夏目」を含むもの

リクエスト URL
https://ndlsearch.ndl.go.jp/api/opensearch?dpid=aozora&creator=%e5%a4%8f%e7%9b%ae
API 3

OpenURL

コンテンツのリンク先を指定するためにメタデータを URL 形式でリンクサーバに送信する規格です。検索結果は HTML（ブラウザ表示） で返戻されます。

基本仕様: NISO Z39.88-2004 (R2010)

📌
返される HTML は本サービスの Web 検索結果画面と同一のため、本サービスの対応ブラウザでのみ表示が可能です。

リクエスト形式

GET https://ndlsearch.ndl.go.jp/api/openurl?genre=[genre]&…
GET https://mina.ndl.go.jp/api/openurl?au=[au]&…

genre 指定と検索先

genre 指定検索対象
genre=article記事レベルの書誌のみ
genre=title / genre=book記事レベル以外の書誌（タイトルレベル・図書/巻号レベル）
（指定なし）すべてのデータ
⚠️
genre（または rft.genre）による指定と atitle/btitle による指定に矛盾がある場合はエラーになります。また、ndl_dpid を指定すると genre によるデータプロバイダ指定は無効になります。

指定可能な検索項目

No. 項目 article article以外 指定なし 一致条件 複数
1aulast（rft.aulast）○○○部分一致×
2aufirst（rft.aufirst）○○○部分一致×
3au（rft.au）○○○部分一致○
4title / atitle（rft.title / rft.atitle）○—○部分一致○
5btitle（rft.btitle）—○○部分一致○
6jtitle（rft.jtitle）○—○部分一致○
7pub（rft.pub）—○○部分一致○
8issn（rft.issn）○○○前方一致×
9isbn（rft.isbn）—○○完全一致 or 前方一致×
10any○○○部分一致○
11ndl_jpno（独自拡張）—○○前方一致×
12ndl_dpid（独自拡張）○○○完全一致○（OR）
13mediatype（独自拡張）○—○完全一致○
ℹ️
項目間の論理条件はすべて AND です。
isbn・issn・ndl_jpno 以外は空白区切りで複数ワードを指定可能（ndl_dpid は OR 検索、他は AND 検索）。
aufirst と aulast を同時に指定した場合は AND 検索。
返戻件数は最大 10,000 件。

リクエスト例

例1: タイトルに「電子図書館」を含むもの

リクエスト URL
https://ndlsearch.ndl.go.jp/api/openurl?btitle=%e9%9b%bb%e5%ad%90%e5%9b%b3%e6%9b%b8%e9%a4%a8

例2: 著者に「夏目漱石」を含むもの

リクエスト URL
https://ndlsearch.ndl.go.jp/api/openurl?au=%e5%a4%8f%e7%9b%ae%e6%bc%b1%e7%9f%b3
API 4

OAI-PMH

OAI（Open Archives Initiative）が定めたサーバ間メタデータ収集プロトコルです。差分ハーベストにより、更新されたメタデータのみを効率的に収集できます。

基本仕様: http://www.openarchives.org/OAI/openarchivesprotocol.html

日本語訳（国立情報学研究所）: https://www.nii.ac.jp/irp/archive/translation/oai-pmh2.0/

リクエスト形式

GET https://ndlsearch.ndl.go.jp/api/oaipmh?verb=[verb]&…

リポジトリ基本情報（Identify レスポンス）

タグ名応答値
repositoryName国立国会図書館サーチ
baseURLhttps://ndlsearch.ndl.go.jp/api/oaipmh
protocolVersionVersion 2.0
deletedRecordpersistent
granularityYYYY-MM-DDThh:mm:ssZ

サポートリクエスト（verb）と引数

No. verb identifier metadataPrefix from until set resumptionToken
1GetRecord ◎◎————
2Identify ——————
3ListIdentifiers —◎○○○○
4ListMetadataFormats ○—————
5ListRecords —◎○○○○
6ListSets —————○

◎：必須、○：任意

metadataPrefix（返戻スキーマ）

  • oai_dc: Dublin Core 形式
  • dcndl: DC-NDL 形式（国立国会図書館拡張）
  • dcndl_v3: DC-NDL ver.3.0（ドラフト版のため注意）
📌
dcndl_v1 を指定した場合、個体情報（dcndl:Item）は 1 メタデータにつき 1 件のみ返戻されます。

identifier 形式

本サービスでは identifier として メタデータ ID を使用します。

形式
oai(ドメイン名):[リポジトリ番号]-[アイテム番号]

例: oai:ndlsearch.ndl.go.jp:R100000002-I000001287827

set（選択的ハーベスト）

set を指定することで特定グループのデータのみを取得できます。

  • データプロバイダ ID
  • データグループ ID
  • コレクションコード（ndl-dl / ndl-dl-online にのみ付与）
  • 公開範囲（Access Rights）（ndl-dl / ndl-dl-online にのみ付与）
⚠️
複数の set を掛け合わせて検索することはできません。また、set を指定して収集している場合、メタデータ上の値が変更されても削除レコードが出力されないため、定期的に全件を収集して洗い替えることを推奨します。

OAI-PMH アクセス制限

大量アクセスによる負荷を回避するため、以下の制限があります。

  • ListRecords および ListIdentifiers では from が必須
  • 指定可能な期間（from〜until）は 最大1年
  • until を省略した場合、from から 1 年後が until とみなされる
  • 一度に返戻される件数は 200件。201件目以降は resumptionToken を使って続きを取得
🕐
本サービスのタイムゾーンは JST です。from・until は JST として処理され、返戻データの datestamp も JST で返されます。

リクエスト例

Identify
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=Identify
ListMetadataFormats
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=ListMetadataFormats
ListIdentifiers（from 指定）
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=ListIdentifiers
  &metadataPrefix=oai_dc
  &from=2024-01-05
ListRecords（青空文庫、from 指定）
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=ListRecords
  &metadataPrefix=oai_dc
  &set=aozora
  &from=2024-01-05
GetRecord（特定レコード取得）
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=GetRecord
  &metadataPrefix=oai_dc
  &identifier=oai:ndlsearch.ndl.go.jp:R100000002-I000001287827
ListSets
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=ListSets
コレクションコードを指定して検索
https://ndlsearch.ndl.go.jp/api/oaipmh?verb=ListIdentifiers
  &metadataPrefix=oai_dc
  &from=2024-01-05

  • *11 ndl-article-online: オンライン資料の記事書誌を横断的に取得可能なデータプロバイダ。zassaku-online・ndl-dl・ndl-dl-online の記事レベル書誌が対象。
  • *12 jpro 系: 近日中に刊行される図書のメタデータ（近刊情報）を含む。jpro-book・jpro-online・jpro-audiobook は jpro の部分集合。

DC-NDL スキーマ

DC-NDL（ダブリンコアメタデータ記述） は、国立国会図書館がダブリンコアを基盤に独自拡張を加えたメタデータ記述形式です。SRU・OpenSearch・OAI-PMH の返戻データで利用されます。
